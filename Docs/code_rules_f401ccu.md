Правила написания кода 
Для микроконтроллера STM32F401CCU6. 

частоту и настройки извлекай из системных переменных, необходимые частоты для делителей из системных переменных
Используй HAL. Все необходимые функции размести в одном файле c/h снабди файл комментариями, также в файле h приведи 
пример использования сразу за заголовком файла, а также код для цикла в main() если он необходим

Все данные включая возвращаемые данные и данные для инициализации, хранить в структуре и передавать 
только через структуру. в структуре инициализации веделить посдструктуру dbg для хранения отладочной информации.
Используй соглашения c99
Все функции, взаимодействующие с HAL, должны проверять возвращаемые значения и корректно 
обрабатывать ошибки, возвращая понятный статус. 
не использовать динамическую память.
функции должны возвращать только статус выполенения.  
 Для переменных, используемых как в main loop, так и в прерываниях, 
необходимо предусмотреть механизмы защиты (критические секции, отключение прерываний). 
 В .h файле, рядом с ключевыми #define, должны быть примеры 
расчета значений. Все необходимые константы должны быть описаны через define в h файле. 
Добавь в заголовочный файл #include "main.h". Использование include guard. Если создается 
драйвер микросхемы, то перенети в define все константы из документации. 
при иницализации использовать константы не оборачивая их. Для иницализации используем константы из h файла

Код должен быть короткий и лакочничный. не создавай функции которые вызываются редко или имеют 
внутри 2-4 строки, вставляй это прямо в код основной процедуры 

веди нумерацию версий в заголовке h файла в формате xx.yy с датой и временем dd.mm.yy hh:mm:ss

Doxygen function documentation comment. Комментарии должны быть на русском в кодировке utf-8. 
В коде и примерах: не использовать последовательность /* или */ нигде, включая Doxygen и примеры; 
все  комментарии оформлять только через // в начале строки.
